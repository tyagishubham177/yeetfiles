# Phase 4: Smart Cleanup and Social Proof

## Goal

Increase decision speed and shareable momentum without turning YeetFiles into a black box.

## Stream 4A: Comparison view

| Task | Deliverable | Verification |
| --- | --- | --- |
| Similar-photo grouping | Burst or near-duplicate groups created from real assets | Inspect grouped results on device |
| Side-by-side comparison card | Two-up layout for hard choices | Visual review and usability test |
| `Keep best` flow | One item kept while others are routed through the chosen action path | Verify resulting statuses |

## Stream 4B: Smart buckets and smart sort refinement

| Task | Deliverable | Verification |
| --- | --- | --- |
| Large-files bucket | High-impact storage candidates grouped clearly | Compare against actual file sizes |
| Better deletability heuristic | Smarter first-session ordering based on screenshots, size, and repeat shots | Manual review of first-session quality |
| Optional blur or low-value signal experiments | Clearly marked as experimental if attempted | Accuracy check on a mixed set |

## Stream 4C: Share and social surfaces

| Task | Deliverable | Verification |
| --- | --- | --- |
| Story-friendly share card | 1080x1920 local PNG summary | Share to story-style targets and inspect |
| Weekly streak tracking | Cross-session streak state | Verify across repeated usage |
| Lifetime stats | Total reviewed and total freed | Inspect after multiple sessions |

## Bifurcation

## Smart ranking confidence

Preferred path:

- use only heuristics users would find intuitive and easy to explain

Fallback path:

- keep smart ranking as an opt-in sort mode if its choices feel too opaque

## Exit gate

Phase 4 is done when the smart features genuinely help users decide faster and the share surfaces still feel tasteful.

## Exit checklist

- comparison view speeds up hard decisions instead of slowing them down
- smart buckets feel helpful, not manipulative
- share card looks intentional in a 9:16 story format
- lifetime stats remain internally consistent with session stats
- no smart feature creates false confidence about what should be deleted
